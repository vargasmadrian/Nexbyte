Política de Privacidad

Generador de Aulas Virtuales — Última actualización: abril 2026

1. Información que recopilamos

La extensión "Generador de Aulas Virtuales" recopila la siguiente información cuando el usuario inicia sesión:

  • Nombre completo y correo electrónico — proporcionados a través de la autenticación con Google OAuth.
  • Foto de perfil — obtenida de la cuenta de Google del usuario.

No recopilamos información personal adicional, datos de navegación ni historial de uso fuera de la plataforma Moodle.

2. Cómo utilizamos la información

La información recopilada se utiliza exclusivamente para:

  • Autenticar al usuario y gestionar su sesión.
  • Asociar su cuenta con su suscripción activa.
  • Registrar el uso de créditos por acción (importar plantillas, generar contenido, etc.).
  • Mostrar el nombre del usuario dentro de la extensión.

3. Almacenamiento de datos

Los datos de autenticación se almacenan de forma segura en Supabase (plataforma backend). Las credenciales de sesión se guardan localmente en el navegador mediante chrome.storage y se eliminan al cerrar sesión.

4. Compartición de datos

No compartimos, vendemos ni transferimos información personal del usuario a terceros. Los datos solo se transmiten entre la extensión y nuestro backend en Supabase para la autenticación y gestión de suscripciones.

5. Permisos de la extensión

  • activeTab — para interactuar con la pestaña de Moodle activa.
  • identity — para autenticación con Google.
  • scripting — para automatizar acciones en Moodle (crear tareas, subir recursos, etc.).
  • storage — para guardar la sesión del usuario localmente.
  • tabs — para detectar la pestaña de Moodle y coordinar acciones.
  • host_permissions (moodle.pucese.edu.ec) — para ejecutar las automatizaciones en la plataforma Moodle.
  • host_permissions (supabase.co) — para comunicarse con el backend de autenticación y suscripciones.

6. Seguridad

Implementamos medidas de seguridad estándar de la industria:

  • Autenticación segura mediante OAuth 2.0 con Google.
  • Comunicación cifrada (HTTPS) con el backend.
  • Row Level Security (RLS) en la base de datos para aislar los datos de cada usuario.

7. Eliminación de datos

El usuario puede cerrar sesión en cualquier momento desde la extensión, lo cual elimina los datos de sesión almacenados localmente. Para solicitar la eliminación completa de su cuenta y datos asociados, puede contactarnos por correo electrónico.
